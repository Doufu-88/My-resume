# 项目二：在线购物车系统

## 需求文档

### 1. 项目概述
构建一个 B2C 在线购物车系统，支持商品浏览、购物车管理、优惠券使用、订单结算。

### 2. 功能需求

#### 2.1 商品浏览
- 商品列表支持分页（默认每页20条），支持按分类、价格区间、关键词筛选
- 商品详情展示：商品名、图片（支持多图轮播）、价格、库存、规格（颜色/尺寸等 SKU）
- 同商品不同 SKU 的价格和库存独立管理
- 商品可设置限购数量（每人最多购买N件）

#### 2.2 购物车管理
- 用户可将商品加入购物车，选择 SKU 和数量
- 购物车中同一 SKU 多次加入时数量叠加
- 数量不能超过库存（下单时再次校验）
- 购物车支持修改数量（加减按钮，最少为1）
- 购物车支持删除商品、批量选择、全选
- 购物车展示：商品图片、名称、SKU信息、单价、数量、小计
- 购物车数据需在用户登录后同步（未登录时使用本地存储）

#### 2.3 优惠券使用
- 支持满减券（满X元减Y元）、折扣券（打Z折）、免邮券
- 每个订单限用1张优惠券（不可叠加）
- 优惠券有有效期限制
- 优惠券有最低消费金额限制
- 部分优惠券限定特定商品/分类可用
- 优惠券使用后不可退还

#### 2.4 订单结算
- 结算时计算：商品总价 - 优惠券抵扣 + 运费 = 实付金额
- 选中购物车商品后进入结算页
- 支持选择收货地址（可新增、编辑、删除地址）
- 支持多种支付方式：余额支付、微信支付、支付宝
- 下单时扣减库存（需保证原子性，防止超卖）
- 下单成功后清空购物车中已购商品
- 下单后30分钟内未支付，订单自动取消并恢复库存

#### 2.5 订单管理
- 订单状态：待付款 → 已付款 → 已发货 → 已收货 → 已完成
- 支持取消订单（仅待付款状态可取消）
- 支持申请退款（已付款/已发货状态）
- 退款后库存恢复
- 订单详情展示：商品信息、金额明细、物流信息、时间线

### 3. 非功能需求
- 库存扣减需保证原子性
- 订单创建需保证幂等性（防重复提交）
- 支付回调需做签名验证
- 接口响应时间 < 300ms

## 已知 Bug 列表

### Bug #1：库存超卖（严重）
- **模块**：订单结算 - 下单
- **现象**：高并发下单时，实际成交数量超过库存量
- **复现步骤**：将某商品库存设为10，用 JMeter 50线程并发下单
- **预期**：最多成交10件
- **实际**：库存扣减未加锁/未用CAS，高并发下可能产生12-15件订单

### Bug #2：优惠券计算顺序错误（严重）
- **模块**：优惠券使用
- **现象**：满减券的金额计算在折扣之后，而非折扣之前
- **复现步骤**：商品总价200元，使用"满150减50"券。对比实付金额。
- **预期**：满减在折前计算（先满减后折扣）
- **实际**：始终先折扣后满减，导致实际优惠金额与预期不符

### Bug #3：退款后优惠券未处理（中等）
- **模块**：订单管理 - 退款
- **现象**：使用优惠券的订单退款后，退款金额包含了优惠券抵扣部分
- **复现步骤**：使用满200减50优惠券下单，实付150元。申请全额退款。
- **预期**：退款150元（实付金额）
- **实际**：退款了商品原价200元，多退了50元

### Bug #4：购物车不同设备不同步（中等）
- **模块**：购物车管理
- **现象**：用户在A设备加购后，B设备登录看不到
- **复现步骤**：两个设备登录同一账号，一个加购后另一个查看
- **预期**：购物车应实时同步
- **实际**：购物车只在登录时同步一次，不实时更新

### Bug #5：收货地址手机号未校验（轻微）
- **模块**：收货地址
- **现象**：可保存手机号为"00000000000"的地址
- **复现步骤**：新增地址时填写无效手机号
- **预期**：校验手机号格式
- **实际**：仅校验了非空

### Bug #6：支付超时订单未自动取消（中等）
- **模块**：订单管理
- **现象**：下单超30分钟未支付，订单仍为"待付款"
- **复现步骤**：下一笔订单不支付，等待超过30分钟
- **预期**：订单状态变为"已取消"，库存恢复
- **实际**：定时任务存在扫描遗漏bug

### Bug #7：部分退款金额计算错误（严重）
- **模块**：订单管理 - 退款
- **现象**：含优惠券的订单部分退款时，金额按原价比例而非实付比例计算
- **复现步骤**：买A(100元)+B(50元)，用满100减20优惠券，实付130元。仅退A。
- **预期**：退款 = 100/150 * 130 = 86.67元
- **实际**：退回了A的原价100元

### Bug #8：SKU切换后数量重置（轻微）
- **模块**：商品详情 - SKU选择
- **现象**：选红色M码数量调为3，切换蓝色L码后数量变回1
- **预期**：切换SKU保留数量
- **实际**：切换SKU时数量被重置
