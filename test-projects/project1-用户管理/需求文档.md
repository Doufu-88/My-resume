# 项目一：用户管理系统

## 需求文档

### 1. 项目概述
构建一个 Web 端用户管理系统，支持用户注册、登录、信息管理、角色权限控制。

### 2. 功能需求

#### 2.1 用户注册
- 用户填写用户名（4-20位字母数字下划线）、密码（8-20位，必须含大小写字母+数字）、邮箱、手机号
- 系统校验用户名唯一性，已存在则提示"用户名已被注册"
- 密码需二次确认，不一致则提示"两次密码输入不一致"
- 邮箱格式校验（xxx@xxx.xxx）
- 手机号格式校验（11位中国大陆手机号，1开头）
- 注册成功后发送验证邮件，跳转至登录页
- 注册成功后用户状态默认为"启用"

#### 2.2 用户登录
- 支持用户名+密码登录
- 连续输错5次，账号锁定30分钟
- 登录成功后生成 JWT Token，有效期2小时
- 支持"记住我"功能，勾选后 Token 有效期延长至7天
- 登录成功后跳转至首页

#### 2.3 用户信息管理
- 用户可修改个人头像（支持 jpg/png/gif，最大5MB）
- 用户可修改昵称、个人简介、手机号
- 修改手机号需短信验证码验证
- 用户可修改密码（需输入旧密码验证）
- 管理员可查看所有用户列表，支持分页、搜索、筛选

#### 2.4 角色权限控制
- 系统包含三种角色：超级管理员、普通管理员、普通用户
- 超级管理员：可管理所有用户、分配角色、删除用户
- 普通管理员：可查看用户列表、不可删除用户、不可修改角色
- 普通用户：仅可管理自己的信息

### 3. 非功能需求
- 密码使用 bcrypt 加密存储
- 接口响应时间 < 500ms
- 支持 1000 并发用户
- 日志记录所有用户操作

## 已知 Bug 列表（用于测试实操）

### Bug #1：注册用户名重复校验存在竞态条件（严重）
- **模块**：用户注册
- **现象**：两个请求同时用同一用户名注册，可能都成功
- **复现步骤**：使用 JMeter 同时对同一用户名发送2个注册请求
- **预期**：只有一个成功，另一个提示"用户名已被注册"
- **实际**：高并发下可能两个都成功，数据库中产生两条相同用户名记录

### Bug #2：登录锁定计数未按账号隔离（严重）
- **模块**：用户登录
- **现象**：用户A输错密码3次后，用户B第一次输错就被锁定
- **复现步骤**：先用账号A连续输错密码，再用账号B尝试登录并故意输错
- **预期**：每个账号独立计数
- **实际**：锁定机制使用了全局计数器，所有账号共享失败计数

### Bug #3：修改密码后旧 Token 未失效（中等）
- **模块**：用户信息管理
- **现象**：用户修改密码后，旧 Token 仍可继续访问接口
- **复现步骤**：登录获取 Token → 修改密码 → 用旧 Token 请求其他接口
- **预期**：修改密码后旧 Token 应立即失效
- **实际**：旧 Token 在有效期内仍可使用

### Bug #4：文件上传未校验真实文件类型（中等）
- **模块**：用户信息管理 - 头像上传
- **现象**：将 .exe 文件后缀改为 .jpg 后仍可上传成功
- **复现步骤**：将任意可执行文件重命名为 avatar.jpg 后上传
- **预期**：服务端应校验文件的真实 MIME 类型
- **实际**：仅检查了文件扩展名，未校验魔术字节

### Bug #5：手机号修改验证码可被绕过（中等）
- **模块**：用户信息管理 - 手机号修改
- **现象**：不输入验证码或输入错误验证码，部分情况下仍可修改手机号
- **复现步骤**：修改手机号时，先输入正确验证码，再快速修改为另一个手机号
- **预期**：每次修改手机号都应独立验证
- **实际**：验证码校验存在60秒窗口缓存，期间可多次修改

### Bug #6：分页参数未做边界校验（轻微）
- **模块**：管理员 - 用户列表
- **现象**：pageSize 设为0或负数时服务端报500错误
- **复现步骤**：请求 `GET /api/users?page=1&pageSize=0`
- **预期**：返回参数错误提示
- **实际**：服务端未处理边界情况，抛出异常

### Bug #7：SQL 注入漏洞在搜索参数中（严重）
- **模块**：管理员 - 用户搜索
- **现象**：搜索框输入 `' OR '1'='1` 返回所有用户数据
- **复现步骤**：在用户搜索框输入 SQL 注入语句
- **预期**：输入应被参数化处理或转义
- **实际**：搜索参数直接拼接到 SQL，存在 SQL 注入风险

### Bug #8：删除用户时未级联删除关联数据（轻微）
- **模块**：管理员 - 删除用户
- **现象**：删除用户后，该用户的日志、角色关联等数据残留在数据库中
- **复现步骤**：删除一个用户，直接查数据库
- **预期**：删除用户时同步清理或标记关联数据
- **实际**：仅删除了 users 表记录
